# Raycast Reference Analysis

Sources: https://www.raycast.com, https://manual.raycast.com/action-panel

## Layout
- Command-first layout centered on speed, keyboard interaction, and focused action surfaces.
- Product storytelling uses launcher panels, command lists, and hotkey cues.

## Navigation
- Primary navigation is simple, but the product model is action search plus contextual menus.
- Action panels and submenus expose power without crowding the default view.

## Typography
- Compact, highly scannable labels with clear command titles and shortcuts.

## Grid
- Command lists use one-column density; marketing pages use bold hero rhythm.

## Components
- Command palette, hotkey badges, action list rows, quick actions, context menus, extension cards.

## Cards
- Cards behave like commands: title, description, icon, shortcut, and action affordance.

## Hero
- Short, memorable headline paired with a command-product visual.

## Motion
- Fast command feedback, crisp hover, keyboard focus, and submenu transitions.

## CTA
- Download/action CTAs are direct and utility-driven.

## Color
- Dark-first interface with bright red/pink accents and high-contrast command panels.

## Icons
- Icons are central to recognizability; use consistent stroke or filled symbols.

## Empty State
- No-results states should suggest commands, filters, or next actions.

## Loading
- Loading should keep the command surface stable.

## Visual Philosophy
- Keyboard-first power, speed, and contextual action.
