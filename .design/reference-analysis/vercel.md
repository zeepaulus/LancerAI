# Vercel Reference Analysis

Sources: https://vercel.com, https://vercel.com/docs/analytics, https://vercel.com/blog/how-we-made-the-vercel-dashboard-twice-as-fast

## Layout
- Developer-grade dashboard language: precise metrics, sparse chrome, strong black/white contrast.
- Analytics surfaces emphasize comparison, time, and performance.

## Navigation
- Product nav is terse; dashboard patterns prioritize project context and deploy/status hierarchy.

## Typography
- Clean, functional sans-serif with strong numeric hierarchy.

## Grid
- Metric cards, charts, tables, and logs sit on a structured grid.

## Components
- KPI cards, chart panels, status rows, logs, project cards, tabular analytics.

## Cards
- Minimal cards with clear borders, strong title/value hierarchy, and dense supporting metadata.

## Hero
- Technical storytelling; product confidence comes from performance and infrastructure framing.

## Motion
- Motion is minimal and performance-minded.

## CTA
- Clear, pragmatic CTAs for deployment, signup, or docs.

## Color
- Monochrome foundation with subtle semantic colors.

## Icons
- Small technical icons; avoid decorative icon overload.

## Empty State
- Explain what data will appear and how to generate it.

## Loading
- Skeletons should match final chart/card layout to avoid layout shift.

## Visual Philosophy
- Precise, fast, trustworthy, metric-led.
