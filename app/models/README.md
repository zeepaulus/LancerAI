# `app/models/` - SQLAlchemy ORM Models

`app/models/` is the source of truth for PostgreSQL schema. Alembic migrations in `migration/` are generated from these models.

## Design Decisions

- SQLAlchemy 2 declarative ORM.
- UUID strings as primary keys.
- Deterministic naming convention for constraints and indexes.
- Timezone-aware timestamps from database defaults.
- JSON columns for flexible AI outputs and reports.
- User-scoped records for CVs, matches and interviews.

## Models

| File | Table | Purpose |
|---|---|---|
| `user.py` | `users` | Accounts, roles, tenant boundary, password hash |
| `cv_record.py` | `cv_records` | Uploaded CV data, extracted/optimized JSON, score/status |
| `job_listing.py` | `job_listings` | Crawled/imported job descriptions |
| `job_match_result.py` | `job_match_results` | Match component scores, missing skills, workflow status |
| `interview_session.py` | `interview_sessions` | Session setup, status, final scores/report metadata |
| `interview_transcript.py` | `interview_transcripts` | Turn-level transcript rows |
| `llm_cache.py` | `llm_response_cache` | Prompt/response semantic cache |
| `base.py` | shared metadata | Declarative base and naming convention |

## Important Columns

### `users`

| Column | Notes |
|---|---|
| `id` | UUID string primary key |
| `tenant_id` | Server-assigned isolation boundary in current MVP |
| `email` | Unique login identifier |
| `display_name` | Visible name and alternate login identifier |
| `password_hash` | bcrypt hash |
| `role` | `user` or `admin` |
| `is_active` | Soft-disable flag |

### `cv_records`

| Column | Notes |
|---|---|
| `user_id` | FK to owner |
| `filename` | Original upload filename |
| `language` | Defaults to `vi` |
| `extracted_data` | Raw text and structured CV JSON |
| `optimized_data` | Output from optimization pipeline |
| `audit_score` | 0-100 score from scorecard |
| `optimization_mode` | Example: `standard`, `roast` |
| `status` | Example: `extracted`, `optimized` |

### `job_match_results`

| Column | Notes |
|---|---|
| `match_score` | Normalized 0-1 overall score |
| `frequency_score` | Keyword overlap component |
| `position_score` | Section-weighted component |
| `semantic_score` | Embedding similarity component |
| `missing_skills` | JSON list |
| `status` | `recommended`, `saved`, `applied`, `rejected` |

### `interview_sessions`

| Column | Notes |
|---|---|
| `job_listing_id` | Optional FK to `job_listings` |
| `mode` | `practice`, `mock`, `quick` |
| `status` | Session lifecycle status |
| `star_scores` | Setup, plan and final score payload |
| `jd_snapshot` | Optional JD text snapshot |
| `tts_voice`, `llm_model` | Runtime provenance |
| `completed_at` | Null until final persistence |

### `llm_response_cache`

| Column | Notes |
|---|---|
| `prompt_hash` | SHA-256 exact lookup |
| `prompt_text`, `response_text` | Debuggable payload |
| `prompt_embedding` | JSON vector |
| `model_name`, `backend` | Provenance |
| `hit_count`, `last_accessed_at` | Cache analytics |

## Relationships

- `User` has many CV records, interview sessions and job matches.
- `CVRecord` has many interview sessions and job matches.
- `JobListing` has many interview sessions and job matches.
- `InterviewSession` has many transcript rows.

## Migration

After model changes:

```bash
uv run alembic revision --autogenerate -m "describe change"
uv run alembic upgrade head
```

Always inspect autogenerated migrations before committing.
