# `migration/` - Alembic

Alembic migrations for the PostgreSQL schema generated from `app/models/`.

The root `alembic.ini` points `script_location` to `migration/alembic/`.

## Commands

```bash
uv run alembic upgrade head
uv run alembic current
uv run alembic history
uv run alembic revision --autogenerate -m "describe change"
```

## Workflow

1. Update ORM models in `app/models/`.
2. Generate migration with `--autogenerate`.
3. Inspect the migration manually.
4. Check indexes, nullable changes, JSON defaults and FK behavior.
5. Run `uv run alembic upgrade head`.
6. Run tests.

## Notes

- Never commit an autogenerated migration without reading it.
- Keep naming deterministic through the convention in `app/models/base.py`.
- The test suite uses SQLite fixtures for many unit tests, but production schema target is PostgreSQL.
